# TeamVenture 一期（小程序）产品设计（UX/UI/UE）

> 目标：把 Web PRD 的核心闭环迁移为小程序形态，并以“短路径、强引导、可对比”为原则降低 HR 决策成本。

---

## 1) 信息架构（IA）

### TabBar（建议）
- `生成方案`（主入口）
- `我的方案`

### 非 Tab 页面
- `登录/授权页`（首登）
- `方案对比页`
- `方案详情页`
- `供应商详情页`（可选：一期也可直接在方案详情里弹层展示联系方式）

---

## 2) 核心用户旅程（Happy Path）

### Journey A：生成并确认方案（一期主闭环）
1. 首页进入“生成方案”
2. Step 1 填写基础信息（人数/预算/日期/出发地）
3. Step 2 选择偏好（活动/住宿/餐饮/特殊需求）
4. 生成中（进度 + 文案反馈）
5. 进入对比页（3 套方案卡片 + 对比表）
6. 进入详情并“确认方案”
7. 在详情页“联系供应商”（拨号/复制微信）并记录

### Journey B：查看历史方案
1. Tab 切到“我的方案”
2. 列表进入详情
3. 可再次联系供应商/再次分享（一期可选）

---

## 3) 页面规格（UI + 交互）

### 3.1 生成方案页（Step 1：基础信息）
**组件**
- 人数：Stepper + 快捷按钮（20/50/100）
- 预算：区间输入（min/max）+ 自动计算人均提示
- 日期：日期区间选择
- 出发地：文本 + 城市选择（一期可简化为城市）

**校验（UE）**
- 人数：`1–500`（可配置）
- 预算：min>0 且 max>=min；超低预算给提示但允许提交（以便教学展示“建议预算”）
- 日期：end>=start；跨度上限（如 1–3 天）一期可限定

**状态**
- Empty：首次进入默认值（人数50、2天1夜）+ 引导文案
- Error：字段下方就地提示

### 3.2 生成方案页（Step 2：偏好）
**组件**
- 活动类型：多选 chips（拓展/休闲/文化/运动）
- 住宿标准：单选（经济/舒适/品质）
- 餐饮偏好：多选
- 特殊需求：textarea（可选）

### 3.3 生成中页（Loading）
**原则（UE）**
- 给用户“系统在做什么”的可见性：匹配供应商→生成方案→校验预算
- 超时策略：超过 60s 给“稍后通知/重试”按钮（一期可只做重试）

### 3.4 方案对比页（核心）
**组件**
- 3 套方案卡片（A/B/C）：人均、亮点、住宿/活动数、总预算
- 对比表：关键差异（预算/住宿/活动/餐饮/交通）
- CTA：
  - 主 CTA：`查看详情`
  - 次 CTA：`重新生成`（回到 Step 2 保留输入）

**交互**
- 默认推荐一套（按“平衡型”）但不替用户决策：推荐标签 + 理由

### 3.5 方案详情页
**模块**
- 概览：人均/总预算/天数/亮点
- 行程：按天折叠面板（避免长页面）
- 预算：按类目折叠（交通/住宿/餐饮/活动/其他）
- 供应商：卡片列表（名称、评分、价格区间、标签、联系方式按钮）

**CTA**
- 主 CTA：`确认此方案`
- 次 CTA：`联系供应商`（每个供应商独立按钮）
- 可选：`分享方案`

**状态机（UE）**
- `draft`：显示“确认”按钮
- `confirmed`：显示“已确认”状态；允许继续联系供应商

---

## 4) 小程序工程实现提示（wechat-minware）

- `setData` 合并更新：对比页与详情页数据较大，避免频繁更新深层对象
- 列表分页：我的方案建议分页加载（pageSize 10）
- 网络层：封装 `request`，统一注入 session token、统一错误码弹窗

---

## 5) 体验验收标准（抽样）

- Given 已填写合法输入，When 点击“生成方案”，Then 60 秒内展示对比页；超时则给可重试提示。
- Given 在对比页，When 点击某方案“查看详情”，Then 进入详情且展示行程/预算/供应商模块，不出现空白块。
- Given 方案为 draft，When 点击“确认此方案”，Then 状态变为 confirmed 且在“我的方案”列表可见。
- Given 在供应商卡片点击“拨号/复制微信”，Then 触发系统能力并上报一次 `supplier_contact` 埋点。
