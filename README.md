# AI Ideas / 产品想法与PRD仓库（One‑Page）

本仓库用于沉淀「AI 原生产品」的想法池、产品文档（PRD/BP）与可复用模板，并配套一套面向产品人的 AI Coding 课程大纲，支持从 0→1 的快速论证与落地。

## 你可以在这里找到什么

1) **想法池（Idea Pool）**
- `ai-product-ideas.md`：AI 产品想法收集与记录模板 + 初始想法集（toC/toB/toProC）。

2) **产品文档（PRD / Business Plan）**
- `prds/ego-echo-workplace-recovery-prd.md`：Ego‑Echo（职场压力复原）微信小程序 MVP 的 PRD（目标/范围/流程/数据结构/API/埋点/合规与隐私等）。
- `prds/ego-echo-business-plan.md`：Ego‑Echo 的 Business Plan（市场、产品、技术方案、质量与安全、单位经济、GTM、风险等）。

3) **可复用模板（Templates）**
- `templates/business-plan-template.md`：0→1 新产品 BP 模板（含 AI & Tech、Eval/Safety、成本与风险表等章节）。

4) **课程内容（Educational Products）**
- `educational-products/README.md`：课程概览（面向产品/设计/非技术背景的 AI 辅助开发）。
- `educational-products/syllabus.md`：12 周详细大纲与实践项目。

## 仓库的核心原则（适用于产品与技术讨论）
- **结构化优先**：输入/输出尽量结构化（便于渲染、评测、埋点、复盘）。
- **可验证**：每个结论尽量配“证据/数据/假设/验证计划”。
- **隐私与安全**：对情绪/心理相关场景默认采用最小化数据与明确边界；危机内容优先安全兜底。
- **可落地**：设计必须能转化为工程任务（API、数据结构、指标、实验计划、里程碑）。

## 推荐阅读路径（10 分钟上手）
1. 先看想法池：`ai-product-ideas.md`
2. 需要写 BP：从模板开始 `templates/business-plan-template.md`
3. 看一个完整样例：`prds/ego-echo-workplace-recovery-prd.md` → `prds/ego-echo-business-plan.md`
4. 想体系化学习：`educational-products/README.md` → `educational-products/syllabus.md`

## 文档维护建议（轻量规则）
- 文件命名尽量稳定（避免破坏链接与目录锚点）。
- PRD/BP 变更建议按“版本号 + 日期 + 变更点”记录在文档头部。
- 新增产品建议在 `prds/` 下按 `产品代号-模块-prd.md` / `产品代号-business-plan.md` 组织。
