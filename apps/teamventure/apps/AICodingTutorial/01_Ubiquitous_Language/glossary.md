# Ubiquitous Language Glossary (领域共同语言术语表)

> **警告**：此表是本项目沟通的唯一标准。在代码命名、数据库字段设计、API 定义中，必须严格使用本表中的英文术语 (English Term)。

## 1. 核心实体 (Core Entities)

| 中文术语 | 英文术语 | 简写 | 定义描述 | 业务规则/约束 |
| :--- | :--- | :--- | :--- | :--- |
| **员工** | `Employee` | - | 公司的内部人员，系统的最终用户。 | 必须拥有唯一的工号 (`employee_id`) 和企业邮箱。 |
| **团队** | `Team` | - | 员工所属的组织架构单元。 | 一个员工只能属于一个主团队。 |
| **活动** | `Activity` | - | 可供预约的活动类型元数据。 | 例如：“羽毛球”、“甚至”、“周五分享会”。包含封面图、描述。 |
| **场次** | `Session` | - | 某个活动在具体时间段的实例化。 | 是预约的直接对象。必须包含：开始时间、结束时间、最大人数、地点。 |
| **预约单** | `Booking` | - | 员工对某个场次的报名记录。 | 一个员工对同一个场次只能有一个有效预约。 |
| **候补** | `Waitlist` | - | 当场次满员时，员工进入的等待队列。 | 先入先出 (FIFO)。当有预约取消时，自动提升第一位。 |

## 2. 状态定义 (Status Definitions)

### 2.1 场次状态 (SessionStatus)
| 状态值 | 描述 | 触发条件 |
| :--- | :--- | :--- |
| `PUBLISHED` | 已发布 | 管理员创建并发布场次。此时可接受预约。 |
| `FULL` | 已满员 | 预约人数达到 `max_capacity`。此时仅接受候补。 |
| `CANCELLED` | 已取消 | 管理员手动取消，或因不可抗力取消。需通知所有已预约员工。 |
| `COMPLETED` | 已结束 | 当前时间 > 结束时间。 |

### 2.2 预约状态 (BookingStatus)
| 状态值 | 描述 | 触发条件 |
| :--- | :--- | :--- |
| `CONFIRMED` | 已确认 | 用户报名成功，且库存充足。 |
| `WAITING` | 候补中 | 场次已满，用户在候补队列中。 |
| `CANCELLED` | 已取消 | 用户主动取消。 |
| `CHECKED_IN` | 已签到 | 用户在现场扫码签到（可选功能）。 |

## 3. 动词与操作 (Verbs & Operations)

| 动作 | 英文动词 | 描述 |
| :--- | :--- | :--- |
| **发布** | `Publish` | 将活动场次对用户可见。 |
| **报名** | `Book` / `Join` | 用户申请参加某个场次。 |
| **取消报名** | `Cancel Booking` | 用户撤销自己的报名。 |
| **核销/签到** | `Check-in` | 确认用户实际到场。 |
| **自动补位** | `Auto-promote` | 系统自动将候补第一名转为已确认状态。 |

## 4. 易混淆概念辨析 (Disambiguation)

*   **Activity vs Session**: 
    *   `Activity` 是**类**（如：羽毛球活动），是静态的介绍。
    *   `Session` 是**对象**（如：本周五晚上的羽毛球局），是动态的，有时效的。
    *   *错误示例*：User books an Activity. 
    *   *正确示例*：User books a Session of an Activity.

*   **Cancel vs Revoke**:
    *   `Cancel` 用于用户取消预约，或管理员取消场次。
    *   `Revoke` 通常用于权限撤销，本项目中不使用。
