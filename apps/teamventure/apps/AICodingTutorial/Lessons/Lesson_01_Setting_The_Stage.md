# Lesson 01: 设定舞台 —— AI 大脑初始化

## 1. 教学目标
本节课结束后，你将学会如何为 AI 设置“工程规则”，确保它生成的每一行代码都符合你的项目规范。

## 2. 为什么需要这一步？
很多人直接问 AI：“帮我写一个 Java 的 Controller”。
AI 会根据它训练数据中的“平均水平”来写。结果可能是：
- 它是用 Java 8 写的，但你需要 Java 17。
- 它把业务逻辑写在了 Controller 里，违反了你的架构。
- 它使用的字段名和你数据库里的对不上。

**这一步的目标是：消除 AI 的随机性。**

## 3. 核心资产
在开始任何代码生成之前，你需要准备好以下两个“宪法”文件：
- `00_Meta_Context/rules_for_ai.md`: 规定 AI 必须做什么，禁止做什么。
- `00_Meta_Context/project_convention.md`: 规定项目的物理定律（技术栈、命名法）。

## 4. 教学实操：如何初始化 AI
当你开启一个新的对话（Chat Session）时，执行以下操作：

### 步骤 A：喂食规则
**你输入：**
> "我正在开发一个企业活动预约系统。请先读取以下两个文件，作为我们后续所有协作的基础。
> 1. [粘贴 rules_for_ai.md 的内容]
> 2. [粘贴 project_convention.md 的内容]
> 请回复：'规则已确认'。"

### 步骤 B：观察 AI 的反应
一个受过良好引导的 AI 应该能够总结出：
- “我知道我要用 Java 17 和 COLA 架构。”
- “我不会在 Controller 里写业务逻辑。”
- “我会严格遵守命名规范。”

## 5. 关键原理解析
### 唯一真理源 (SSoT)
在 AI Coding 中，我们不希望 AI “发挥”。我们希望它是一个 **“精准的翻译器”**。通过设定 rules，你把 AI 从“创作模式”切换到了“工程模式”。
