## 单元0.1：案例项目介绍 - TeamVenture（5分钟）

**讲师**：

“在开始一切技术准备之前，我们先来看看，通过这门课程，我们最终要做出一个什么样的产品。

这就是我们的实战项目——**TeamVenture**。

它的产品定位是：一款AI团建策划助手，旨在帮助公司的HR或行政人员，快速、智能地生成高质量的团建活动方案。

它的核心功能非常清晰：

1.  **需求收集**：用户输入团建的人数、预算、城市、以及风格偏好等。

在本课程中，我们将重点完成前端小程序的核心功能开发。至于后端的服务和部署，我们会在后续的进阶课程中覆盖。

## 单元0.2：课程介绍与学习方法（5分钟）

**讲师**：

“了解了我们要做的产品，接下来我们来谈谈这门课独特的学习方法。它和传统的编程课非常不同，我们遵循一个四步循环：**观察、理解、验证和迭代**。

1.  **观察**：你需要仔细看我是如何向AI描述产品需求，以及AI是如何根据这些需求生成代码的。
2.  **理解**：你的重点不是记住代码长什么样，而是去理解每一段代码背后所实现的业务逻辑是什么。比如，这段代码是为了实现‘用户登录’，那段代码是为了‘显示活动列表’。
3.  **验证**：我们会把AI生成的代码，立刻在真实的环境中运行起来，检查功能是否和我们预想的一致。
4.  **迭代**：如果结果不完美，这很正常。你需要学习如何调整和优化你对需求的描述，引导AI给出更精准的答案。这就像与一位聪明的同事沟通，你需要清晰地表达你的想法。

那么，学习这门课，你‘**不需要**’做什么呢？

*   你**不需要**死记硬背任何编程语法。
*   你**不需要**理解复杂的技术底层原理。
*   你**不需要**有任何编程经验，零基础就是最好的起点。

你需要做的，恰恰是你最擅长的事情：

*   **清晰地描述产品需求**。
*   **验证功能是否符合预期**。
*   以及，**学会如何与AI高效沟通**。

好，作为本单元的第一个验证点，请大家思考一下：‘AI辅助开发’和‘传统编程学习’，这两者最核心的区别是什么？你可以在我们的课程交流群里分享你的看法。”

## 单元0.3：VPN与网络环境配置（10分钟）

**讲师**：

“接下来，我们进入第一个也是最关键的动手环节：配置网络环境。

很多同学在这一步会遇到困难，但请放心，我们把这个过程拆解得非常细致。首先，我们要明确一个概念：现在的网络服务通常由两部分组成——**‘机场服务商’**（提供线路订阅）和**‘客户端软件’**（负责连接）。

你需要先购买一个‘机场服务’，获取一个**订阅链接**（通常以 `http` 开头），然后把它填入你的客户端软件中。

下面我们分平台手把手教学：

### 1. Windows 平台

*   **推荐软件**：v2rayN 或 Clash Verge。
*   **操作步骤**：
    1.  下载并解压软件（注意：不要放在含中文的路径下）。
    2.  打开软件，找到‘订阅’或‘Profiles’菜单。
    3.  点击‘添加订阅’，把你在机场购买的订阅链接粘贴进去，点击更新。
    4.  在节点列表中，右键选择一个节点（推荐香港或新加坡节点），设为‘活动节点’。
    5.  **关键一步**：在系统托盘（右下角）找到软件图标，右键选择‘系统代理’ -> ‘自动配置系统代理’。图标变色即代表开启成功。

### 2. Mac 平台

*   **推荐软件**：ClashX Pro 或 Clash Verge。
*   **操作步骤**：
    1.  下载 `.dmg` 安装包并安装。
    2.  首次打开需要输入开机密码授权。
    3.  点击状态栏的小猫图标，选择‘配置’ -> ‘托管配置’ -> ‘管理’。
    4.  点击‘添加’，粘贴订阅链接。
    5.  **关键一步**：点击状态栏图标，勾选‘设置为系统代理’。同时，建议在‘出站模式’中选择‘规则判断’，这样访问国内网站不会变慢。

### 3. iOS (iPhone/iPad) 平台

*   **难点**：中国区 App Store 无法下载相关软件。
*   **解决方案**：你需要一个**非中国区（如美区）的 Apple ID**。
    *   *方法A*：淘宝购买（有风险，仅限下载用）。
    *   *方法B*：自己注册（推荐，需准备一个未注册过 Apple ID 的邮箱）。
*   **推荐软件**：Shadowrocket（俗称‘小火箭’，需付费，约 $2.99）。
*   **操作步骤**：
    1.  打开 App Store，点击右上角头像，滑到底部‘退出登录’。
    2.  登录美区 Apple ID。
    3.  搜索并下载 Shadowrocket。
    4.  打开小火箭，点击右上角 `+` 号，类型选择 `Subscribe`（订阅）。
    5.  粘贴订阅链接，点击保存。
    6.  回到首页，开启顶部的开关，首次连接会弹出 VPN 权限确认，点击‘Allow’并输入锁屏密码。

### 4. Android 平台

*   **推荐软件**：Clash for Android 或 v2rayNG。
*   **操作步骤**：
    1.  下载 `.apk` 安装包（通常机场官网会提供下载链接）。
    2.  打开软件，点击‘配置’ -> ‘新配置’ -> ‘从 URL 导入’。
    3.  粘贴订阅链接，点击右上角保存。
    4.  回到主界面，点击灰色的启动按钮。
    5.  系统会弹出‘网络连接请求’，点击‘确定’。

**最后，一个通用的验证方法**：
无论你用哪个平台，配置完成后，请打开浏览器访问 `https://claude.ai`。如果能秒开，恭喜你，你的网络环境已经准备就绪！如果打不开，请检查是否开启了‘系统代理’或‘全局路由’模式。”

## 单元0.4：Git版本控制基础（5分钟）

**讲师**：

“我们继续下一个工具的准备：Git。你可以把Git想象成一个‘代码时光机’。

在开发过程中，我们的代码会不断修改。有了Git，你就可以在任何时候，让你的项目‘穿越’回任意一个历史版本，再也不用担心代码改错或丢失了。同时，它也是未来团队协作开发的基础。

我们只需要理解几个核心概念，不需要记忆命令，因为AI会帮我们处理。

| 概念 | 类比 | 说明 |
|---|---|---|
| Repository（仓库） | 项目文件夹 | 存放你所有代码和文件的地方 |
| Commit（提交） | 存档点 | 你为当前项目状态创建的一个快照或存档 |
| Branch（分支） | 平行世界 | 允许你同时开发多个功能而不互相干扰 |
| Pull/Push（拉/推） | 上传/下载 | 与云端的代码仓库同步你的本地代码 |

安装Git非常简单：

*   **Mac用户**：打开‘终端’，输入 `git --version`，如果系统提示你安装，按照指引操作即可。
*   **Windows用户**：请搜索并下载‘Git for Windows’进行安装。

在课程中，像 `git add`, `git commit` 这些常用命令，Claude Code都会自动帮我们执行，我们只需要理解它在做什么就行。

**本单元的验证点**：打开你的终端或命令行工具，输入 `git --version`，如果能看到版本号显示出来，就说明你安装成功了。”

## 单元0.5：Claude Code工具介绍与GitHub关联（10分钟）

**讲师**：

“现在，让我们来认识一下本课程的绝对主角——Claude Code。

Claude Code是Anthropic公司开发的一款顶尖的AI编程助手。它的强大之处在于，能够深刻理解我们的自然语言需求，并直接生成高质量、可执行的完整代码。

它的核心能力包括：
1.  **代码生成**：根据需求直接写代码。
2.  **代码解释**：为你解释看不懂的代码。
3.  **问题修复**：分析错误并给出修复方案。
4.  **代码重构**：优化代码，使其更简洁高效。

安装Claude Code需要Node.js环境。安装好Node.js后，在终端执行：`npm install -g @anthropic-ai/claude-code`。

**关键步骤：关联 GitHub**

安装只是第一步，为了让 Claude Code 能够帮我们管理代码，我们必须授权它访问我们的 GitHub 仓库。

1.  **初始化**：在终端输入 `claude` 并回车。
2.  **登录授权**：首次运行时，它会提示你进行登录。按回车键，浏览器会自动打开 Anthropic 的登录页面。登录你的账号。
3.  **GitHub 授权**：登录成功后，终端会询问是否授权 GitHub 访问权限。
    *   它会显示一个链接，点击或复制到浏览器打开。
    *   在 GitHub 授权页面，点击绿色的 **'Authorize Anthropic'** 按钮。
    *   **注意**：如果你的 GitHub 开启了双重验证（2FA），可能需要输入验证码。
4.  **验证连接**：授权完成后，回到终端，你会看到 `Login successful` 的提示。

此时，Claude Code 就已经拿到了你 GitHub 的‘钥匙’，它可以帮你拉取代码、创建新仓库、甚至直接推送代码更新。

**本单元的验证点**：在终端输入 `claude`，如果它不再提示登录，而是直接显示 `Claude Code >` 的交互提示符，说明关联成功！”

## 单元0.6：Manus介绍与GitHub关联（5分钟）

**讲师**：

“除了Claude Code，我们还有一个非常强大的全能型AI助手——Manus。

Manus不仅仅是一个编程工具，它更像是一个全能的数字员工。它拥有独立的运行环境，可以像真人一样操作浏览器、使用各种工具、甚至执行复杂的任务规划。

在我们的产品开发过程中，Manus可以扮演多种角色：
1.  **产品经理助理**：调研竞品、整理需求。
2.  **数据分析师**：收集数据、生成报表。
3.  **全栈工程师**：搭建环境、部署服务。

**关键步骤：关联 GitHub**

为了让 Manus 能够帮我们把做好的产品部署上线，或者把调研报告存入仓库，我们也需要给它 GitHub 的权限。

1.  **进入设置**：打开 Manus 的主界面，点击左下角的‘设置’（Settings）图标。
2.  **找到集成选项**：在设置菜单中，找到 **'Integrations'**（集成）或 **'Connected Apps'**（已连接应用）。
3.  **连接 GitHub**：
    *   找到 GitHub 图标，点击右侧的 **'Connect'**（连接）按钮。
    *   在弹出的 GitHub 授权窗口中，确认授权给 Manus。
    *   你可以选择授权所有仓库（All repositories），也可以只选择特定的仓库（Only select repositories）。为了方便后续课程，建议选择 **'All repositories'**。
4.  **确认状态**：连接成功后，按钮状态会变为 **'Connected'**（已连接），并显示你的 GitHub 用户名。

这样，Manus 就正式成为了你团队中的一员，它不仅能思考，还能直接帮你‘干活’——比如把写好的文档直接 Push 到你的仓库里。

**本单元的验证点**：在 Manus 的设置页面中，确认 GitHub 状态显示为‘已连接’。”
