## 单元 0.2：VPN与网络环境配置（5分钟）

### 开场引入

同学们好，这节课我们来解决一个最基础但非常关键的问题——网络环境配置。

你可能会问：为什么学编程还要配置网络？

原因很简单：我们这门课要用的核心工具 Claude Code，它是由美国的 Anthropic 公司提供的。就像你想看 Netflix 需要特殊的网络环境一样，使用 Claude Code 也需要能够访问国际网络。

**【画面提示：显示 claude.ai 网站】**

### 核心讲解

配置 VPN 其实就像给你的电脑装一个"翻译官"。

正常情况下，你的电脑直接和目标网站对话，但有些网站听不懂"中国话"。VPN 就是帮你找一个"中间人"，你先跟中间人说，中间人再帮你转达给目标网站。

**【画面提示：动画演示 VPN 工作原理】**

市面上有很多 VPN 服务，我这里不做具体推荐，因为每个人的情况不一样。如果你是个人学习，可以搜索"机场服务"，价格大概在每月 20 到 50 元左右。如果你在公司，可以咨询 IT 部门，他们可能已经有企业 VPN 方案。

### 操作演示

假设你已经有了一个 VPN 服务，接下来的配置有三个关键点：

**第一点：选择全局代理模式**

**【画面提示：VPN 软件设置界面，标注"全局代理"选项】**

很多 VPN 软件有两种模式：一种叫"PAC 模式"或者"智能模式"，另一种叫"全局模式"。

PAC 模式就像一个聪明但有时候犯糊涂的门卫，它会自己判断哪些网站需要走 VPN，哪些不需要。问题是，它有时候会判断错。

全局模式就简单粗暴——所有流量都走 VPN，虽然可能会稍微慢一点，但绝对不会出错。

对于我们学编程来说，**请选择全局模式**，避免后面遇到莫名其妙的网络问题。

**第二点：验证配置是否成功**

**【画面提示：浏览器打开 claude.ai】**

打开浏览器，访问 `https://claude.ai`。如果你能看到一个对话界面，恭喜你，配置成功了。

如果显示"无法访问此网站"，说明 VPN 没有生效，检查一下是否开启了全局模式，或者换一个节点试试。

**第三点：确保终端也能访问**

**【画面提示：打开终端，输入命令】**

这一步很重要但容易被忽略。浏览器能访问不代表终端也能访问。

什么是终端？就是那个黑色或白色的命令行窗口，程序员用来输入指令的地方。我们后面安装 Claude Code 就要在终端里操作。

大多数 VPN 软件在全局模式下会自动代理终端流量。如果你发现终端不行，需要在 VPN 软件里找一下"允许局域网连接"或者"系统代理"这类选项，把它打开。

### 常见问题

**连接不稳定怎么办？**

这是最常见的问题。VPN 节点就像不同的中转站，有些站人多拥挤，有些站畅通无阻。

**【画面提示：VPN 节点列表】**

如果你发现网速很慢或者经常断开，试着换一个节点。根据我的经验，日本和新加坡的节点通常比较稳定，美国节点有时候会比较慢。

### 验证检查点

在进入下一节之前，请确认：

- [ ] 浏览器能正常打开 `https://claude.ai`
- [ ] VPN 设置为全局代理模式

好，网络环境配置完成。这是最基础的一步，后面所有工具的安装都依赖这个网络环境。如果这里卡住了，不要着急往后学，先把网络问题解决。

## 单元 0.3：Git 版本控制基础（5分钟）

### 开场引入

接下来我们来认识一个程序员世界里"人人必备"的工具——Git。

先说一个你肯定经历过的场景：

你写了一份方案，保存为"方案v1"。老板看完说改几个地方，你改完保存为"方案v2"。然后老板又说，还是第一版好，但是要加上第二版的某个部分...

**【画面提示：显示一堆文件名："方案v1"、"方案v2"、"方案v2-改"、"方案v2-改-最终版"、"方案v2-改-最终版-真的最终"】**

是不是很熟悉？这就是没有版本控制的痛苦。

Git 就是为了解决这个问题而生的。它是代码世界的"时光机"，让你可以随时回到任何一个历史版本。

### 核心概念

**【画面提示：概念对照表动画】**

Git 有几个核心概念，我用大家熟悉的东西来类比，**你不需要记住这些，只需要有个印象**：

**Repository（仓库）**——就是一个文件夹

只不过这个文件夹被 Git "监控"着，里面的任何变化都会被记录下来。

**Commit（提交）**——就是游戏里的存档点

每次你觉得代码写到一个阶段了，就"存个档"。以后想回到这个状态，随时可以读档。

**Branch（分支）**——就是平行世界

比如你想尝试一个新功能，但又怕把现有的代码搞坏。这时候你可以开一个"分支"，在分支里随便折腾，主世界不受影响。折腾成功了，再把分支合并回来。

**Pull/Push（拉取/推送）**——就是下载和上传

你的代码可以存在云端（比如 GitHub），Pull 就是把云端的代码下载到本地，Push 就是把本地的代码上传到云端。

### 安装验证

**【画面提示：打开终端】**

现在我们来确认一下你的电脑有没有安装 Git。

**如果你用的是 Mac：**

打开终端，输入：

```
git --version
```

**【画面提示：终端显示 git version 2.x.x】**

如果显示类似 `git version 2.39.0` 这样的版本号，说明已经安装好了。Mac 系统通常自带 Git。

**如果你用的是 Windows：**

同样打开命令行（按 Win+R，输入 cmd），输入 `git --version`。

如果显示"不是内部或外部命令"，说明没有安装。这时候你需要去 `git-scm.com` 下载 Git for Windows 并安装。

**【画面提示：Git 官网下载页面】**

安装过程一路点"下一步"就行，不需要改任何设置。

### 常用命令预览

这里我先给你"剧透"几个最常用的 Git 命令，**不需要现在记住**，后面用到的时候你自然就会了：

**【画面提示：命令列表动画】**

- `git status` — 看看当前有什么变化（相当于"查看游戏进度"）
- `git add .` — 把所有变化加入暂存区（相当于"选择要存档的内容"）
- `git commit -m "说明"` — 存档，并写一句话描述这次改了什么
- `git push` — 上传到云端
- `git pull` — 从云端下载最新版本

**这些命令后面 Claude Code 会帮你执行**，你只需要知道它们是干什么的就行。

### 验证检查点

在进入下一节之前，请确认：

- [ ] 终端输入 `git --version` 显示版本号

Git 的概念就介绍到这里。记住，**你不需要成为 Git 专家**，这门课的核心是用 AI 帮你写代码，Git 只是帮你保存和管理这些代码的工具。

## 单元 0.4：Claude Code 工具介绍（5分钟）

### 开场引入

好，重头戏来了。这节课我们要认识这门课的"主角"——Claude Code。

Claude Code 是 Anthropic 公司推出的 AI 编程助手。你可以把它想象成一个 24 小时待命的"程序员搭档"。你告诉它你想实现什么功能，它就帮你写代码。

**【画面提示：Claude Code 终端界面】**

### 核心能力

Claude Code 能做什么？简单说，它有四大超能力：

**第一，代码生成**

你用中文描述你想要什么，它帮你写出可运行的代码。比如你说"帮我写一个登录页面"，它就能生成登录页面的完整代码。

**第二，代码解释**

看不懂别人写的代码？把代码丢给它，它会用通俗的语言解释每一行在干什么。

**第三，Bug 修复**

程序报错了？把错误信息丢给它，它帮你定位问题并修复。

**第四，代码重构**

代码能跑但写得很乱？它可以帮你整理成更清晰、更规范的版本。

### Claude Code vs Cursor

**【画面提示：对比表格】**

你可能听说过另一个工具叫 Cursor。我们这门课两个都会用到，但定位不同：

| 特性 | Claude Code | Cursor |
|------|-------------|--------|
| 工作方式 | 在终端里对话 | 在编辑器里对话 |
| 擅长场景 | 项目级别的大任务 | 单个文件的小修改 |
| 本课程角色 | **主力工具** | 辅助工具 |

打个比方：

Claude Code 就像一个"项目经理级别的程序员"，你跟它说"帮我搭建一个小程序后端"，它能帮你从头到尾把整个项目架构搭起来。

Cursor 更像一个"随叫随到的代码助手"，你在写代码的时候，可以随时问它"这一行怎么写"。

**这门课我们主要用 Claude Code**，因为我们的目标是从零到一搭建一个完整的产品。Cursor 会在后面做一些细节调整的时候用到。

### 安装步骤

**【画面提示：终端界面】**

安装 Claude Code 需要先确保你的电脑有 Node.js 环境。我们在前置课程已经安装过了，这里直接开始安装 Claude Code。

打开终端，输入以下命令：

```
npm install -g @anthropic-ai/claude-code
```

**【画面提示：终端执行安装命令】**

这行命令的意思是"用 npm（Node.js 的包管理器）全局安装 Claude Code"。

安装过程可能需要一两分钟，取决于你的网络速度。如果卡住不动，检查一下 VPN 是否正常。

### 验证安装

安装完成后，输入：

```
claude --version
```

**【画面提示：显示版本号】**

如果显示类似 `1.0.x` 的版本号，恭喜你，安装成功了！

### 首次使用

第一次使用 Claude Code 需要登录你的 Anthropic 账号。输入：

```
claude
```

**【画面提示：Claude Code 启动界面，显示登录提示】**

它会引导你通过浏览器完成登录授权。登录成功后，你就可以开始和 Claude Code 对话了。

我们先不深入使用，下一个 Lesson 会正式开始用它来搭建项目。

### 验证检查点

在进入下一节之前，请确认：

- [ ] 终端输入 `claude --version` 显示版本号
- [ ] （可选）尝试输入 `claude` 并完成登录

Claude Code 就介绍到这里。记住，**它是你的 AI 编程搭档**，从下一课开始，我们几乎所有的代码都会通过和它对话来完成。

## 单元 0.5：微信小程序基础概念（5分钟）

### 开场引入

最后一节课，我们来了解一下这门课要开发的产品形态——微信小程序。

什么是小程序？简单说，它就是一个**不需要下载安装、在微信里直接运行的轻量级应用**。

**【画面提示：微信小程序入口演示】**

你打开微信，下拉一下，就能看到最近使用的小程序。或者在聊天窗口里，有人分享一个小程序卡片，你点一下就能打开使用。

为什么选择小程序作为我们课程的实战项目？

**第一，开发门槛相对较低**，非常适合零基础学习。
**第二，用户获取成本低**，用户不需要下载 App，在微信里就能用。
**第三，微信生态庞大**，你做的产品能直接触达 12 亿微信用户。

### 技术架构概览

**【画面提示：小程序项目结构图】**

一个小程序项目长什么样？我给你展示一下它的文件结构：

```
小程序项目/
├── app.json          ← 全局配置文件
├── app.js            ← 全局逻辑文件
├── pages/            ← 页面目录
│   └── index/        ← 首页
│       ├── index.wxml    ← 页面结构（像 HTML）
│       ├── index.wxss    ← 页面样式（像 CSS）
│       └── index.js      ← 页面逻辑
└── utils/            ← 工具函数
```

**不需要记住这些！** 我只是让你有个整体印象。

简单说：
- `.wxml` 文件定义页面上有什么元素（按钮、文字、图片）
- `.wxss` 文件定义这些元素长什么样（颜色、大小、位置）
- `.js` 文件定义这些元素的行为（点击按钮会发生什么）

### 本地存储 API

**【画面提示：代码示例】**

小程序有一个非常实用的功能——本地存储。它能让你在用户的手机上保存一些数据，下次打开小程序时还能读取到。

我们的"AI 团建策划助手"就会用到这个功能，比如保存用户之前生成过的方案。

核心就三个 API：

```javascript
// 保存数据
wx.setStorageSync('key', value)

// 读取数据
const data = wx.getStorageSync('key')

// 删除数据
wx.removeStorageSync('key')
```

**这些代码你也不需要记**，到时候 Claude Code 会帮你写。我只是让你知道小程序有这个能力。

### 开发工具安装

**【画面提示：微信开发者工具官网】**

开发小程序需要一个专门的工具——**微信开发者工具**。

这个工具是微信官方提供的，免费的，去官网下载安装就行：

`https://developers.weixin.qq.com/miniprogram/dev/devtools/download.html`

**【画面提示：下载页面，选择对应系统版本】**

### 创建测试项目

安装完成后，打开微信开发者工具，用微信扫码登录。

**【画面提示：开发者工具首页】**

然后点"新建小程序项目"：

**【画面提示：新建项目界面，标注各个选项】**

点击"确定"，稍等几秒，你就能看到一个带有示例代码的小程序项目了。

**【画面提示：开发者工具主界面，左侧模拟器显示 Hello World】**

左边是模拟器，显示小程序运行效果。中间是代码编辑区。右边是调试工具。

你可以点击模拟器里的按钮玩一玩，感受一下小程序的交互。
